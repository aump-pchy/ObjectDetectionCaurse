2
แตกไฟล์ ตรวจสอบโครงสร้าง dataset
แตก .zip แล้วตรวจดูให้ครบก่อนอัปโหลด — โครงสร้างต้องตรงตามนี้:
Dataset/ ├── test/ │ ├── images/ ← ภาพทดสอบ │ └── labels/ ← annotation ทดสอบ ├── train/ │ ├── images/ ← ภาพสำหรับเทรน (ส่วนใหญ่) │ └── labels/ ← annotation เทรน ├── valid/ │ ├── images/ ← ภาพสำหรับ validate │ └── labels/ ← annotation validate └── data.yaml ← config file (สำคัญมาก!)
⚠️
ชื่อไฟล์ภาพกับ .txt ต้องตรงกัน — img001.jpg ↔ img001.txt และค่าใน .txt ต้องเป็น normalized 0–1 เสมอ
3
ตรวจสอบไฟล์ data.yaml ✅ สำคัญมาก
ไฟล์ config บอก path ของ dataset + จำนวน class + ชื่อ class — ต้องถูกต้อง 100% ก่อนเทรน
# data.yaml — ตัวอย่าง structure ที่ถูกต้อง path: /content/dataset # root folder บน Colab train: train/images # ตำแหน่ง train images val: valid/images # ตำแหน่ง validation images test: test/images # ตำแหน่ง test images nc: 3 # จำนวน class names: ['rose', 'tulip', 'daisy'] # ชื่อ class ตามลำดับ # roboflow metadata (ข้ามได้) roboflow: workspace: my-workspace project: my-project version: 1 license: CC BY 4.0
🚨
ข้อผิดพลาดที่พบบ่อย: path ใน data.yaml อาจยังเป็น path บนเครื่อง local อยู่ — บน Colab ต้องเปลี่ยน path: เป็น /content/dataset

🔥 โหลด YOLO11n + เทรน ⚠️ มี note
โหลด pretrained YOLO11n แล้วเทรนบน custom dataset
python · Cell 4 — Train YOLO11n
from ultralytics import YOLO

# โหลด pre-trained YOLO11n (ดาวน์โหลดอัตโนมัติ ~6MB)
model = YOLO('yolo11n.pt')

# เทรนโมเดล
results = model.train(
    data='/content/dataset/data.yaml',  # ✅ ต้องเป็น absolute path
    epochs=50,       # รอบการเทรน (ดูหมายเหตุด้านล่าง)
    imgsz=640,       # ขนาดภาพที่ใช้เทรน
    batch=16,        # จำนวนภาพต่อ batch (T4 GPU รับได้)
    device=0,        # 0=GPU, 'cpu'=CPU
    project='/content/runs',
    name='workshop5',
    exist_ok=True
)

print(f"✅ Training complete!")
print(f"Best model: /content/runs/workshop5/weights/best.pt")
⚠️
50 epochs สำหรับทดสอบเท่านั้น — ผลอาจยังไม่ดีพอ ถ้าต้องการ mAP สูงแนะนำ 100–150 epochs สำหรับ dataset เล็ก (~200–500 ภาพ) / 300+ epochs สำหรับ final model
💡
เพิ่ม batch parameter — ใน PPTX ไม่มี แต่สำคัญ: batch=16 เหมาะกับ T4 GPU บน Colab — ถ้า OOM ให้ลดเป็น 8

Code สำหรับประเมินโมเดล
python · Evaluate model (val set)
from ultralytics import YOLO

# โหลด best model
model = YOLO('/content/runs/workshop5/weights/best.pt')

# ประเมินบน validation set
metrics = model.val(data='/content/dataset/data.yaml')

print(f"mAP50:    {metrics.box.map50:.4f}")   # IoU=0.5
print(f"mAP50-95: {metrics.box.map:.4f}")    # IoU=0.5:0.95
print(f"Precision: {metrics.box.mp:.4f}")
print(f"Recall:    {metrics.box.mr:.4f}")
📏 Metrics ที่ต้องรู้
mAP
Mean Average Precision
ค่าหลักที่บอกความแม่น — ยิ่งสูงยิ่งดี
P
Precision
จาก detect ทั้งหมด ถูกกี่ % (TP/TP+FP)
R
Recall
จากวัตถุจริงทั้งหมด จับได้กี่ % (TP/TP+FN)
IoU
Intersection over Union
bbox ทับกันกับ ground truth กี่ %
📌
mAP50 vs mAP50-95 — mAP50 วัดที่ IoU threshold 0.5 (ง่ายกว่า) / mAP50-95 วัดเฉลี่ย 10 threshold (0.5:0.05:0.95) — เป็น metric มาตรฐาน COCO ที่เข้มกว่า

2
โหลดโมเดล → ตรวจจับ → แสดงผลด้วย OpenCV
python · Detect + Display with OpenCV
from ultralytics import YOLO
import cv2
from matplotlib import pyplot as plt

# โหลดโมเดลที่เทรนใน Workshop 5-1
model = YOLO('/content/runs/workshop5/weights/best.pt')

# รัน inference
results = model(img_filename, conf=0.35, imgsz=640)
result  = results[0]

# แสดงผลจำนวนที่ detect ได้
print(f"พบวัตถุ: {len(result.boxes)} ชิ้น")
for box in result.boxes:
    label = model.names[int(box.cls)]
    conf  = float(box.conf)
    x1, y1, x2, y2 = [int(v) for v in box.xyxy[0]]
    print(f"  ✓ {label:15s}  conf:{conf:.2f}  box:[{x1},{y1},{x2},{y2}]")

# แสดงภาพผลด้วย matplotlib (สำหรับ Colab)
img_result = result.plot()           # วาด bbox + label บนภาพ
img_rgb    = img_result[..., ::-1]  # BGR → RGB

plt.figure(figsize=(12, 8))
plt.imshow(img_rgb)
plt.axis('off')
plt.title(f'YOLO11n Custom — {len(result.boxes)} objects detected')
plt.tight_layout()
plt.show()

# บันทึกภาพผล
cv2.imwrite('result.jpg', result.plot())
files.download('result.jpg')

JOB 5 — ส่งจริง
โปรแกรมรู้จำใบหน้าเพื่อน

แบ่งกลุ่ม กลุ่มละ 5 คน — เทรน YOLO11n เท่านั้น

1
เก็บข้อมูลภาพใบหน้า — ถ่ายภาพสมาชิกในกลุ่มให้หลากหลาย หลายมุม หลายแสง หลายระยะ คนละอย่างน้อย 50–100 ภาพ
2
ติดป้ายกำกับ (Label) — วาด bounding box รอบใบหน้า ระบุ class เป็นชื่อสมาชิก (5 class = 5 คน)
3
จัดเตรียม dataset — แบ่ง train/val/test เหมาะสม เพิ่ม augmentation ผ่าน Roboflow
4
เทรนโมเดล YOLO11n เท่านั้น — ใช้ yolo11n.pt เป็น pretrained weight เท่านั้น ห้ามใช้ yolo11s/m/l/x
5
ประเมินประสิทธิภาพ — บันทึก mAP50 จาก val set
6
ทดสอบจริง — ทดสอบแยกสมาชิกกลุ่มออกจากเพื่อนทั้งชั้นเรียน — นับว่าจำถูกกี่คน
คะแนน = (mAP ÷ 20) + จำนวนที่แยกถูกต้อง
mAP 80% → 4 คะแนน + แยกถูก 4/5 คน → +4 คะแนน = 8 คะแนน
💡 Tips สำหรับ Face Detection โดยเฉพาะ
📸
เก็บภาพหลากหลาย: หน้าตรง, ด้านข้าง, เงียน, มีแว่น, มีหมวก, แสงต่างๆ — โมเดลต้องเจอ variation ระหว่างเทรน
🏷️
Label ให้แน่น: bbox ต้องครอบแค่ใบหน้า ไม่รวมคอหรือไหล่ — bbox ที่หลวมทำให้โมเดล confused
⚖️
Balance ภาพทุก class: แต่ละคนควรมีจำนวนภาพใกล้เคียงกัน — ถ้าคนหนึ่งมี 200 แต่อีกคนมี 20 โมเดลจะ bias
🔢
เพิ่ม epochs: face detection ต้องการอย่างน้อย 100 epochs — 50 epochs มักยังไม่ converge
